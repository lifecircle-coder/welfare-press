# THE복지 pSEO 킬러콘텐츠 개발 계획안 (v2.0)

## 🎯 프로젝트 개요
공공데이터 API의 부실함을 극복하고, MZ세대의 검색 의도(Intent)에 정확히 부합하는 100개 이상의 '살아있는' 지역 복지 페이지를 완전 자동화로 구축함.

---

## 🏗️ 1. 데이터 파이프라인: "프로그래매틱 데이터 팩토리"
기존의 단순 API 호출 방식을 버리고, **[수집 -> 정제 -> 강화 -> 저장]**의 4단계 팩토리 구조로 변경합니다.

### Phase 1: Raw Data Collection (수집)
- **대상**: 온라인청년센터(청년정책), 보조금24(중앙부처), 서울시 열린데이터 광장.
- **방식**: 매일 새벽 03:00 정기 배치(Batch) 작업 실행.

### Phase 2: AI Data Enrichment (정제/강화)
- **AI 엔진**: Gemini 1.5 Flash (Free Tier 활용, 비용 0원).
- **수행 작업**:
    - 긴 텍스트에서 `지원금액(숫자)`, `나이제한`, `신청마감일(YYYY-MM-DD)` 추출.
    - 공무원 말투를 2026년 MZ 문체로 자동 요약 및 톤앤매너 변경.
    - SEO용 메타 설명(Meta Description) 및 키워드 Tag 자동 생성.

### Phase 3: Knowledge Gap Filling (공백 메우기)
공공 API 정보가 부족할 때, 다음 서브 데이터를 자동으로 병합(Merge)합니다.
- **지역 기반 통계**: "이 지역 청년 10명 중 3명이 이미 확인했습니다." (데이터: 서울 열린데이터)
- **부동산 연관성**: "이 정책은 {관악구} 평균 월세의 25%를 충당할 수 있습니다." (데이터: 한국부동산원)
- **지역 뉴스**: THE복지 내 해당 지역 최신 기사 피드 연동.

---

## 🎨 2. UI/UX 전략: "Zapier-style Matrix Page"
단순 리스트가 아닌, 사용자가 '나에게 맞는 정책'임을 즉각 인지하는 레이아웃을 적용합니다.

### 📱 페이지 구성 (100개 지역별 커스텀)
- **Hero**: `[지역명] 청년이 2026년 4월 현재 당장 받을 수 있는 돈 1,250,000원` (총합 자동 계산)
- **Urgency Card**: D-Day가 가장 짧은 정책 3개를 상위에 배치 ('마감 임박' 배지).
- **Comparison Table**: 혜택 종류별(현금/바우처/서비스) 깔끔한 비교 표.
- **Social Proof**: 지역별 조회수 및 관심도 스태츠(Stats) 노출.

---

## ⚙️ 3. 운영 및 확장성 (Scalability)
대표님이 1만 개 페이지를 관리할 수 있게 만드는 자동화 장치입니다.

### ① AI 퀄리티 스코어 시스템
- 데이터가 부족하거나 오래된 페이지는 시스템이 0~100점의 점수를 부여.
- 60점 이상의 페이지만 Google Search Console에 노출(Index) 승인.
- 40점 미만 '주의 페이지'만 별도 리포트로 보고하여 운영 부하 감소.

### ② SEO 파이프라인
- `Next.js ISR`을 활용하여 데이터 변경 시 1시간 내 포털 검색 결과 자동 업데이트.
- Schema.org의 `FAQPage` 및 `HowTo` 구조화 데이터를 삽입하여 검색 결과 상단(Featured Snippet) 선점.

---

## 🧪 4. 검증 계획
- **기술 검증**: 1개 지역(관악구)을 대상으로 AI 데이터 정제 스크립트 정확도 테스트 (목표 95% 이상).
- **SEO 검증**: 10개 샘플 페이지를 우선 배포 후, 1주일 내 네이버/구글 색인(Index) 여부 확인.
