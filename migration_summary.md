# 🚀 가비아 클라우드 이전 준비 요약 (2026-04-13)

대표님, 나중에 "가비아 클라우드 이전 상세 실행 계획서 작성해"라고 말씀하시면 이 파일을 바탕으로 즉시 세부 플랜을 수립하겠습니다.

## 1. 이전 목적
- **비용 절감**: pSEO 활성화 시 예상되는 트래픽 증가로 인한 Supabase/Vercel의 높은 비용(최대 40만원+)을 가비아 클라우드 고정비(7~9만원) 수준으로 낮춤.

## 2. 현재 환경 vs 가비아 클라우드
| 구분 | 현재 (Supabase + Vercel) | 이전 후 (가비아 클라우드 Gen2) |
|---|---|---|
| 요금 체계 | 유동적 (트래픽 비례 상한 없음) | 고정적 (시간제/월정액, 4TB 국내 트래픽 무료) |
| 기술 구조 | BaaS/PaaS (관리 편함) | IaaS (직접 서버 관리 필요) |
| 배포 방식 | GitHub Push 자동 배포 | **GitHub Actions** 기반 자동 배포 (Jey가 구축 예정) |

## 3. 핵심 작업 예정 사항
- **인증 시스템 교체**: Supabase Auth ➔ NextAuth.js (가장 큰 작업)
- **DB 이전**: Supabase PostgreSQL ➔ 가비아 로컬 PostgreSQL (pg_dump 사용)
- **서버 환경 구축**: Ubuntu + Nginx + PM2 + Node.js (Jey가 가이드 및 자동화 예정)
- **배포 파이프라인**: GitHub Actions를 통해 기존과 동일한 `git push` 경험 유지.

## 4. 진행 시 주의사항
- **오버레이 없는 이전**: 가비아 쪽에 병렬 운영 환경을 먼저 완벽히 구축 및 테스트한 후, 최종 시점에 DNS만 전환하여 서비스 중단 최소화.
- **2026년 8월 이벤트**: 가비아 약정 이벤트 활용 시 추가 할인 가능.
