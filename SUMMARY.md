# 📑 프로젝트 아키텍처 및 안전 작업 가이드 (SUMMARY.md)

이 문서는 프로젝트의 현재 상태를 기록하고, 향후 수정 작업 시 **시스템의 안정성을 100% 보장하기 위한 엄격한 규칙**을 정의합니다. 새로운 대화창에서 작업을 시작할 때 이 파일을 통해 컨텍스트를 즉시 파악할 수 있도록 합니다.

---

## 🛡️ 안전한 작업을 위한 3대 원칙 (Safety Protocols)

새로운 수정사항을 반영할 때, 조 팀장은 다음 원칙을 반드시 준수합니다.

### 1. "무영향성 검증" (No-Impact Verification)
- **사전 분석**: 코드를 수정하기 전, 해당 변경이 기존 기능(특히 발행일 보존, 정렬 성능)에 영향을 주지 않는지 논리적으로 분석하여 보고합니다.
- **대안 제시**: 위험 요소가 발견되면 숨기지 않고 안전한 작업 방식을 우선 제안합니다.

### 2. '깐 대리'의 엄격한 전수조사 (Regression Testing)
- **수정 확인**: 요청한 사항이 정확히 반영되었는지 테스트합니다.
- **기존 기능 유지**: 메인 페이지, 상세 페이지, 관리자 목록이 정상인지 전수 조사합니다.
- **고정값 검증**: 이미 해결된 '발행일(date) 보존' 및 '인덱스 성능'이 그대로 유지되는지 확인합니다.

### 3. '롤백 지점' 명시 및 스냅샷 보존 (Rollback Strategy)
- **작업 전 백업**: 수정 전 완벽한 상태를 Git 커밋 및 로컬 스냅샷으로 보존합니다.
- **즉각 원복**: 0.1%라도 이상이 생기면 1초 만에 가장 완벽했던 이전 상태로 되돌릴 것을 약속합니다.

## 🚀 워크플로우 및 도구 활용
1. **Branch 활용**: `main` 브랜치는 항상 동작하는 상태로 유지하며, 모든 수정은 `feat/` 또는 `fix/` 브랜치에서 진행 후 검증된 것만 합칩니다.
2. **Atomic Updates**: 한 번에 하나씩, 가장 중요한 것부터 고치고 완벽하게 작동하는지 확인 후 다음 단계로 넘어갑니다.
3. **Implementation Plan**: 작업 전 '구현 계획서'를 통해 예상 영향도와 절차를 미리 공유합니다.
4. **뇌피셜 코딩 금지 및 MCP 무기고 활용 (Zero-Hallucination Policy)**
**프레임워크 문법**: Next.js 및 Supabase 관련 코드를 작성/수정하기 전, 반드시 Context7 MCP를 호출하여 최신 공식 문서와 API 레퍼런스를 먼저 확인한다
.
**DB 스키마 보호**: 백엔드 수정 시 원시 SQL 문을 맹목적으로 실행하지 말고, Supabase MCP를 통해 읽기 전용(Safe type) 쿼리로 현재 스키마를 먼저 확인한다
.
**UI/UX 수정**: 시각적인 디자인 요소 추가 시 백엔드 로직을 건드리지 말고 Stitch MCP를 호출하여 시각적 렌더링 로직을 비즈니스 로직과 철저히 격리한다
.
5. **결론 도출 전 검증 (Verify-Before-Conclude)**
모든 버그 수정 및 새로운 기능 추가 지시를 받으면, 코드를 즉시 토해내지 않는다
.
반드시 Sequential Thinking 스킬을 가동하여 수정 사항이 기존 의존성(발행일 보존 등)에 파급 효과(Ripple effect)를 일으키지 않는지 교차 검증하고 가설을 세운다
.

## 🔧 핵심 아키텍처 및 기술 규칙
- **Tech Stack**: Next.js 14+ (App Router), Supabase (PostgreSQL), TypeScript, Tailwind CSS.
- **Date Integrity (중요)**: 
    - `date` 필드는 기사 발행 시점의 고유값으로, 기사 수정 시 절대 변하지 않아야 합니다.
    - 관리자 UI에서 발행일은 `readOnly` 상태를 유지합니다.
- **Session Isolation (세션 격리)**:
    - 홈페이지(Public)와 관리자(Admin) 세션의 충돌을 방지하기 위해 `lib/services.ts`는 반드시 '클라이언트 주입(Client Injection)' 패턴을 사용합니다.
    - 관리자 페이지 및 기능 호출 시에는 항상 명시적으로 `adminSupabase` 클라이언트를 주입하여 공용 세션과의 오염을 차단합니다.
- **Performance**: `created_at` 인덱스(`idx_articles_created_at`)가 생성되어 있으며, 대량 Base64 이미지 본문 삽입은 지양합니다.
- **UI Standard**: 모든 날짜와 시간은 초 단위(`toLocaleString('ko-KR')`)까지 명확히 표시합니다.

## 🎨 로고 및 심볼 제작 진행 현황 (나 과장 인수인계)

**1. 심볼 렌더링 방식 확정**
- 기존의 단순 Vector SVG 변환 시도는 복잡한 3D 입체감, 투명도(Alpha), 다중 그라데이션 매핑을 완벽히 담아내지 못해 폐기되었습니다.
- **해결책 제안**: 고해상도 투명 PNG 원본(`symbol4.png`)에 **파이썬(Python) 기반의 "고급 그라데이션 맵(Gradient Map)" 스크립트** (`high_end_render_v4.py`)를 적용하여 블루 톤 다중 색상을 명도에 맞춰 정밀하게 렌더링하는 방식을 확정했습니다.
- **산출물 도출**: 딥 네이비부터 스카이 블루까지 이어지는 완벽한 입체감과 투명도를 보존한 `symbol4_premium_blue.png` 결과물 생성 완료.

**2. 심볼 + 텍스트 결합 계획 (하이브리드 SVG)**
- 렌더링된 3D 심볼 이미지(`symbol4_premium_blue.png`)와 로고 텍스트("**THE 복지**")를 하나의 파일로 무손실 결합하기 위해 **"하이브리드 SVG(Hybrid SVG)"** 방식을 채택합니다.
- SVG 내부에 `<image>` 태그로 심볼을 (Base64 등) 배치하고, `<text>` 태그로 벡터 텍스트를 배치해 폰트, 컬러, 크기를 코드로 언제든 제어할 수 있게 만듭니다.
- **다음 세션 목표**: 새로운 대화창에서 이 하이브리드 SVG 코드를 작성 및 최적화하여 웹사이트 헤더 사이트 로고 및 파비콘 등에 일괄 적용합니다.
